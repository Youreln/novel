**✨ 纯静态零成本小说管理与阅读系统 \| 基于 GitHub API 驱动 \| 无需后端、无需服务器**

## 🎯 核心功能特性

### 1\. 沉浸式阅读系统

- 支持 **Markdown / TXT** 双格式完美解析渲染

- 三套阅读主题：**白天模式 / 护眼模式 / 夜间模式**

- 自定义字号、行距，所有设置**本地永久缓存**

- 玻璃拟态 UI、圆角卡片、悬浮动效、全站自适应移动端

- 专属书本 Favicon 标签图标，界面精致简洁

### 2\. 云笺全能编辑器

内置全套专业文本排版工具，满足小说创作全需求：

- **基础排版**：加粗、斜体、下划线、删除线、H1/H2/H3 标题、有序/无序列表、对齐方式

- **文本美化**：文字颜色、背景色、清除格式、批量去空行/去空格

- **格式转换**：全半角互换、标点互换、大小写转换、繁简互换

- **实用工具**：查找替换、富文本/纯文本复制、一键导出 TXT、清空编辑区

- 不依赖文件名、时间排序，支持**完全人工自定义排序**

- 通过 `sort.json` 持久保存排序规则，全站统一生效

- 自动兼容新文件，新增小说自动后置，不打乱原有排序

- 排序失败自动降级兜底，保证页面永不空白

### 4\. 云端文件管理能力

- 无需本地 Git 操作，前端一键**新增 / 修改 / 删除**仓库小说

- 自动解析小说元数据，剥离冗余注释，纯净渲染正文

## ⚙️ 系统底层核心原理

本项目为 **纯前端静态架构**，无后端服务器、无数据库，全程依托**GitHub Open API** 实现完整读写与持久化能力。

### 1\. 整体架构分工

|层级|文件|作用说明|
|---|---|---|
|展示层|index\.html|调用公开API拉取小说、读取排序配置、渲染书架与阅读页面|
|操作层|read\.html / Cloudpad|带Token授权API，实现小说新增/修改/删除/排序配置上传|
|数据层|GitHub 仓库|`novel/` 存小说文件，`sort.json` 存排序规则，永久持久化|

### 2\. GitHub 只读 API 读取原理（用户阅读）

普通用户访问书架无需任何权限，调用 GitHub 公开接口：

```Plain Text
https://api.github.com/repos/用户名/仓库名/contents/novel
```

**读取流程：**

1. 前端请求目录接口，获取所有文件元数据（名称、类型、下载地址、sha指纹）

2. 自动筛选 `.md` / `.txt` 有效小说文件

3. 通过 download\_url 拉取原始文本内容

4. 自动剥离自定义元数据注释，清洗正文

5. marked\.js 解析 MD 富文本，TXT 自动规整换行渲染

**只读特性**：匿名访问每小时 60 次限流，无权限、仅可读、不可篡改。

### 3\. GitHub 授权 API 写入原理（管理员编辑）

所有**新增/修改/删除/保存排序**操作，均依赖 **GitHub Token** 授权：

请求头携带身份认证：`Authorization: token 个人Access令牌`

#### 新增小说原理

1. 编辑器接收正文、文件名、元数据

2. 前端拼接标准化文件内容

3. 内容 **Base64 编码**（GitHub API强制规范）

4. 调用创建文件API，携带Token完成云端新建

#### 修改小说原理

1. 读取旧文件 **sha 唯一指纹**

2. 拉取原文、剥离元数据供编辑

3. 新内容重新 Base64 编码

4. 携带旧 sha 值覆写文件，防止版本冲突

#### 删除小说原理

1. 获取目标文件 sha 值

2. 授权API校验权限后云端永久删除

### 4\. 云笺编辑器专属底层原理（read\.html 源码深度解析）

本项目编辑器并非通用模板，为 Youreln 自研**云笺Cloudpad定制版**，深度适配小说仓库管理，拥有专属的元数据解析、编码容错、排序兜底、双端编辑、离线缓存机制，完美适配GitHub API运行规范。

#### 4\.1 双编辑器互通原理

项目配备两套功能完全同步的编辑入口，底层逻辑一致、数据互通：

- **在线独立编辑器**：https://youreln\.github\.io/Cloudpad/，纯独立页面，功能最全，无页面布局限制；

- **站内内置编辑器**：项目根目录 `read.html`，小说站顶部「写小说」按钮一键跳转，适配仓库专属配置；

两套编辑器共用同一套 GitHub Token 授权逻辑、文件解析规则、排序算法，操作数据实时同步同一仓库，无功能差异。

#### 4\.2 Markdown 渲染与 CDN 容错原理

编辑器依赖外部公共CDN加载 `marked.min.js` 实现MD解析，针对境外CDN访问失败问题做了兼容适配：

- 正常网络：加载官方CDN资源，实现完整 Markdown 实时预览、标题解析、富文本渲染；

- 网络异常（境外网页无法访问）：预览模块自动降级失效，**纯文本编辑、文件上传、排序管理功能完全不受影响**，保证核心编辑功能永续可用；

- 编辑区采用 `pre-wrap` 原生排版，无JS依赖也可正常换行、保存文本格式。

#### 4\.3 专属小说元数据（ZW\_META）运行原理

这是本项目**独家核心机制**，区别于普通静态文本编辑器，专门适配小说阅读个性化配置持久化：

- **存储形式**：以隐藏注释 `<!-- ZW_META:{} -->` 嵌入小说文件头部，不影响正文阅读、不污染渲染效果；

- **存储内容**：自动记录单本小说专属配置——字号、行距、背景色、阅读主题、小说标题、作者信息；

- **解析逻辑**：编辑器载入文件时，正则精准匹配元数据并自动剥离，仅展示纯净正文供编辑；保存时自动重新生成并拼接元数据；

- **渲染逻辑**：阅读页自动读取该注释配置，单本小说独立记忆阅读样式，区别于全局统一设置。

#### 4\.4 全链路编码容错原理（解决中文/特殊文件名乱码）

针对 GitHub API 对中文、特殊字符文件名的兼容问题，源码内置多层解码容错机制：

- 文件读取：采用 `decodeURIComponent`二次解码，彻底解决中文文件名乱码、加载失败问题；

- 内容编解码：兼容两套 Base64 解码逻辑，优先使用 `escape+unescape` 适配中文，异常时自动降级原生解码；

- 页面渲染：排序列表自动转义特殊字符，杜绝XSS漏洞与页面布局错乱；

- JSON容错：排序配置读取异常、格式损坏时，自动初始化空数组，不阻塞页面运行。

#### 4\.5 智能排序同步底层算法（源码核心优化）

编辑器排序并非简单数组排序，拥有**增量同步、去重清理、新旧兼容**智能算法：

1. **存量保留**：同步文件时，优先保留用户手动调整的原有排序顺序，不重置自定义排序；

2. **增量追加**：仓库新增的小说文件，自动追加至排序列表末尾，不打乱已有排序结构；

3. **失效清理**：自动过滤已删除、不存在的失效文件条目，自动去重，保证排序列表干净有效；

4. **空值兜底**：无 sort\.json 配置、配置损坏、接口报错时，自动读取仓库原生文件列表初始化排序，永不空白。

#### 4\.6 富文本编辑工具实现原理

所有排版工具均为前端原生JS实现，无第三方插件依赖，实时编辑、实时预览：

- **选区编辑机制**：通过 `selectionStart / selectionEnd` 获取鼠标选中内容，精准替换格式化标签（加粗、斜体、标题、列表等）；

- **批量文本处理**：内置正则批量去空行、去多余空格、格式清除，一键规整小说排版；

- **格式转换引擎**：原生映射表实现全半角、标点、大小写互换，繁简转换预留拓展接口；

- **本地导出机制**：通过 Blob 二进制对象生成本地TXT文件，完整保留元数据与正文格式。

#### 4\.7 文件增删改精准控制原理

- **新增文件**：清空原有sha指纹与表单信息，全新生成文件内容与编码，提交新建请求；

- **修改文件**：强依赖文件唯一 **sha指纹** 校验，精准匹配云端旧文件，防止多人操作覆盖冲突；

- **删除文件**：二次弹窗确认，先校验文件存在性再执行删除，杜绝无效请求与误删；

- **状态实时反馈**：所有操作自带状态提示，3秒自动消失，成功/错误状态精准区分。

### 5\. 自定义书架排序核心原理

系统**不依赖系统默认排序**，完全由 `novel/sort.json` 控制展示顺序，结合编辑器智能同步算法实现稳定自定义排序。

#### 排序配置结构

```Plain Text
{
  "order": ["第一章.md", "第二章.md", "番外篇.txt"]
}
```

#### 排序执行逻辑

1. 首页优先读取 sort\.json 配置；

2. 按配置数组顺序优先渲染有序小说；

3. 未录入配置的新文件自动追加末尾；

4. 无配置/配置损坏时，自动降级为原生文件顺序，保证不报错。

#### 保存排序逻辑

1. 编辑器前端手动调整数组顺序；

2. 生成标准 JSON 排序文件；

3. Base64 编码覆盖上传 sort\.json；

4. 全站即时生效，所有用户统一看到自定义排序。

## ⚠️ 项目注意事项与限流说明

### 1\. GitHub API 限流规则

- **匿名访问（普通用户）**：每小时 60 次请求，频繁刷新会触发限流、列表空白

- **Token授权（管理员）**：每小时 5000 次请求，无惧高频编辑排序

### 2\. 使用规范

- 个人 Token 严禁公开泄露，避免仓库被恶意篡改删除

- 仅支持 `.md` / `.txt` 格式小说，其他格式无法解析

- 禁止手动篡改 sort\.json 格式，否则排序错乱失效

- 国内网络访问GitHub存在概率超时，加载失败可刷新重试

- 修改内容后请使用 `Ctrl+Shift+R` 强制刷新清除缓存

- 低配设备可关闭页面粒子动效，提升运行流畅度

- 系统自带容错降级机制，异常情况下自动恢复默认展示，保证可用
